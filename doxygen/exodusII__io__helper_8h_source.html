58 // Through several subsequent levels of includes, the exodusII.h
59 // header includes <errno.h>. If <cstring> is subsequently included
60 // in the same translation unit, there is an issue with GCC 5.2.0+ on
61 // OSX which somehow causes errno_t to then be undefined. This
62 // typedef, which is controlled by an autoconf test, seems to fix the
63 // issue and should be safe, since errno_t should have the same type as
64 // errno, which according to the C99 standard is a "macro that expands
65 // to a modifiable lvalue that has type int".
